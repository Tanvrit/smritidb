# Cross-binding conformance corpus

`golden.json` is the bit-exact contract every Smritidb implementation must satisfy. Generated by the TypeScript reference at `packages/core-ts`; verified by every other binding (Rust, Python, Kotlin, Swift) as part of their test suite.

If you're implementing a new binding, you pass conformance iff:

- For every `random_hv` entry, calling your `randomHv(utf8-encoded seed, dim)` and hashing the output bytes with SHA-256 produces the same `sha256` value listed.
- For every `encode_string` entry, your `encodeString(input, dim)` hashes to the same `sha256`.
- For every `similarity_pairs` entry, your `similarity(randomHv(a_seed), randomHv(b_seed))` lands within `1e-9` of `expected`.
- For every `bind_round_trip` entry, `bind(bind(a, b), b)` equals `a` exactly (similarity = 1.0).
- For every `bundle` entry, your `bundle([randomHv(s) for s in seeds])` hashes to the same `sha256`.
- For every `text_bag_of_words` / `text_char_ngrams` entry, the encoder output hashes to the same `sha256`.

Hashes use SHA-256 over the raw byte representation of the hypervector (one byte per bit; values 0 or 1).

## Regenerating

```bash
cd packages/core-ts && pnpm build
node -e "..." > ../../tests/conformance/golden.json
```

The corpus is intentionally small — eight random_hv vectors, three encode_string vectors, three similarity pairs, one bind round-trip, two bundles, two bag-of-words, two char-ngrams. A passing corpus does not prove the full implementation is correct, only that it agrees with the reference at these specific inputs.
